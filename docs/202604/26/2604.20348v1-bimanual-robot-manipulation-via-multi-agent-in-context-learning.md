## 摘要
大语言模型（LLMs）已成为具身控制的强大推理引擎。特别是，上下文学习（ICL）使现成的纯文本 LLM 能够在不进行任何特定任务训练的情况下预测机器人动作，同时保留其泛化能力。然而，将 ICL 应用于双臂操作仍然具有挑战性，因为高维联合动作空间和紧密的手臂间协调约束会迅速超出标准上下文窗口的负荷。为此，我们引入了 BiCICLe（双臂协调上下文学习），这是第一个使标准 LLM 能够在无需微调的情况下执行少样本双臂操作的框架。BiCICLe 将双臂控制建模为一个多智能体主从（leader-follower）问题，将动作空间解耦为顺序的、有条件的单臂预测。这自然地扩展到了“手臂辩论”（Arms' Debate）——一个迭代优化过程，并引入了第三个“LLM 作为评委”（LLM-as-Judge）来评估并选择最合理的协调轨迹。在 TWIN 基准测试的 13 个任务中进行的评估显示，BiCICLe 实现了高达 71.1% 的平均成功率，比表现最好的免训练基准方法高出 6.7 个百分点，并超过了大多数监督学习方法。我们进一步展示了该方法在面对新任务时强大的少样本泛化能力。

## Abstract
Language Models (LLMs) have emerged as powerful reasoning engines for embodied control. In particular, In-Context Learning (ICL) enables off-the-shelf, text-only LLMs to predict robot actions without any task-specific training while preserving their generalization capabilities. Applying ICL to bimanual manipulation remains challenging, as the high-dimensional joint action space and tight inter-arm coordination constraints rapidly overwhelm standard context windows. To address this, we introduce BiCICLe (Bimanual Coordinated In-Context Learning), the first framework that enables standard LLMs to perform few-shot bimanual manipulation without fine-tuning. BiCICLe frames bimanual control as a multi-agent leader-follower problem, decoupling the action space into sequential, conditioned single-arm predictions. This naturally extends to Arms' Debate, an iterative refinement process, and to the introduction of a third LLM-as-Judge to evaluate and select the most plausible coordinated trajectories. Evaluated on 13 tasks from the TWIN benchmark, BiCICLe achieves up to 71.1% average success rate, outperforming the best training-free baseline by 6.7 percentage points and surpassing most supervised methods. We further demonstrate strong few-shot generalization on novel tasks.

### 1. 核心问题与整体含义（研究动机和背景）
双臂机器人操作（如抬起托盘、拧开瓶盖）要求两臂在位置、姿态和力量上实现高度同步。相比单臂控制，双臂操作面临两大挑战：
*   **高维动作空间**：联合动作空间随手臂数量呈指数级增长（单臂 $Z^7 \rightarrow$ 双臂 $Z^{14}$）。
*   **严苛的协调约束**：手臂间的微小误差会迅速累积导致任务失败。
现有的具身大模型（VLA）通常需要海量任务特定数据进行微调。虽然**上下文学习（In-Context Learning, ICL）**允许 LLM 仅通过少量示例完成新任务，但直接将双臂动作序列化会导致 Token 长度激增，超出 LLM 的上下文窗口处理能力，导致计划不连贯。本研究旨在探索如何利用 LLM 的推理能力，在**无需微调**的情况下实现高效的双臂协调控制。

### 2. 方法论：BiCICLe 框架
BiCICLe（Bimanual Coordinated In-Context Learning）的核心思想是将双臂控制解耦为多智能体协作问题。

*   **核心架构：领航者-跟随者分解（Leader-Follower Decomposition）**
    *   **阶段 1（Leader）**：指定一只手臂为 Leader，根据环境观测预测其完整的单臂动作轨迹（$Z^7$ 空间）。
    *   **阶段 2（Follower）**：Follower 智能体在观测环境的同时，将 Leader 的预测轨迹作为额外输入，预测自身的协调动作。这种解耦将 $Z^{14}$ 的预测压力降回了 $Z^7$。
*   **关键技术细节**：
    *   **动作离散化**：将工作空间划分为 $100^3$ 的体素网格，旋转角度以 $5^\circ$ 为间隔离散化，夹持器状态为二进制。
    *   **手臂辩论（Arms’ Debate）**：一种迭代优化机制。Leader 和 Follower 轮流根据对方最新的计划重新规划，通过多轮“对话”消除时空冲突。
    *   **Best-of-N 与 LLM 评判器**：并行生成多个候选轨迹，引入第三个 LLM 充当“法官”，根据碰撞风险、目标匹配度、夹持逻辑等准则对轨迹评分并选出最优解。
*   **观测表示**：将 3D 物体质心坐标序列化为文本字典，避免了直接处理原始图像带来的感知瓶颈。

### 3. 实验设计
*   **数据集/场景**：主要在 **TWIN benchmark**（基于 RLBench 的双臂扩展版）上进行，包含 13 个具有挑战性的双臂任务（如推箱子、移交物品、抬托盘、拉直绳子等）。
*   **对比方法**：
    *   **免训练基准**：RoboPrompt（单智能体/独立双智能体）、KAT（基于关键点的 ICL）、VLM-LF（基于视觉输入的领航者模式）。
    *   **监督学习方法（参考上限）**：ACT、PerAct 2、$\pi_0$-keypose、AnyBimanual、3DFA（当前 SOTA）。
*   **泛化性测试**：设计了两个基准测试之外的新任务（Close Jar, Take Item Out of Box），对比 3DFA 在极少样本下的表现。

### 4. 资源与算力
*   **模型后端**：主要实验使用 **GPT-5-mini**。为了验证框架的通用性，还使用了开源模型 **Qwen 2.5 7B** 和 **Qwen 2.5-VL 7B**。
*   **算力说明**：由于该方法基于 ICL（免训练），因此**不涉及 GPU 训练时长**。
*   **推理延迟**：文中记录了推理时间，BiCICLe 基础版每回合中位耗时约 125.3s，Best-of-N 模式由于并行化处理，耗时约 149.7s。

### 5. 实验数量与充分性
*   **实验规模**：每个任务在 3 个随机种子下进行 100 次测试，总计进行了数千次评估实验。
*   **消融实验**：非常充分。研究者分别对“领航者手臂分配（左/右）”、“辩论机制 vs. 普通对话”、“观测表示（是否含旋转）”、“点云提取算法”进行了详细对比。
*   **客观性**：通过在不同规模（商业 vs. 开源）的模型上复现，证明了方法的架构优势而非仅仅依赖特定模型的强大性能。

### 6. 主要结论与发现
*   **性能卓越**：BiCICLe + Best-of-N 实现了 **71.1%** 的平均成功率，比最强的免训练基准高出 6.7%，甚至在多个任务上超过了经过大量训练的监督学习模型（如 $\pi_0$ 和 PerAct 2）。
*   **解耦的必要性**：实验证明，单智能体（SA）擅长对称同步任务，独立双智能体（DA）擅长非对称分工任务，而 BiCICLe 的主从模式能兼顾两者。
*   **泛化能力强**：在面对完全未见过的新任务时，BiCICLe 的成功率（54.5%）远高于仅用 10 个样本微调的监督学习模型（10.0%）。

### 7. 优点
*   **零微调成本**：无需梯度更新，仅靠 Prompt 即可适配新任务，极大地降低了双臂机器人的部署门槛。
*   **架构优雅**：通过主从分解巧妙解决了 LLM 处理高维动作空间时的“维度灾难”。
*   **鲁棒性优化**：引入“辩论”和“评判”机制，有效抑制了 LLM 生成动作时的随机性和不稳定性。

### 8. 不足与局限
*   **精度限制**：由于采用了体素化（$100^3$）和离散化旋转，对于需要极高精细度的操作（如抓取细绳、滑动笔记本）仍存在物理精度瓶颈。
*   **感知依赖**：实验依赖于仿真环境提供的地面真值掩码（Ground-truth masks），在真实世界中需要更强大的开放词汇目标检测和 3D 感知系统支持。
*   **长程任务挑战**：受限于 LLM 的上下文窗口，无法在 Prompt 中放入过多的演示示例，限制了其在极长步骤任务中的表现。
