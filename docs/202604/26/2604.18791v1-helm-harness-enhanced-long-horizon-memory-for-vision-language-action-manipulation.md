## 摘要
视觉-语言-动作（VLA）模型尽管在短程任务中表现强劲，但在长程操控任务中却存在系统性失效。我们表明，在当前的反应式执行设置下，仅靠延长上下文长度无法解决这一失效问题；相反，它源于执行循环中反复出现的三个缺陷：记忆鸿沟（memory gap）、验证鸿沟（verification gap）和恢复鸿沟（recovery gap）。我们提出了 HELM，这是一个与模型无关的框架，通过三个组件解决这些缺陷：一个情景记忆模块（EMM），通过 CLIP 索引的关键帧检索关键任务历史；一个学习型状态验证器（SV），根据观测、动作、子目标和基于记忆的上下文在执行前预测动作失败；以及一个 Harness 控制器（HC），负责执行回滚和重新规划。SV 是核心的学习贡献：它始终优于基于规则的可行性检查和集成不确定性基准，且其有效性关键取决于对情景记忆的访问。在 LIBERO-LONG 基准上，HELM 将任务成功率相比 OpenVLA 提升了 23.1 个百分点（从 58.4% 提升至 81.5%），而将上下文窗口延长至 H=32 仅带来 5.4 个百分点的提升，且在相同预算下的 LoRA 适配仍比 HELM 低 12.2 个百分点。HELM 还提升了在 CALVIN 上的长程表现，并显著增强了在受控扰动下的恢复成功率。消融实验和机制分析分离了各组件的贡献，我们发布了 LIBERO-Recovery，这是一个用于评估长程操控中故障恢复的扰动注入协议。

## Abstract
Vision-Language-Action (VLA) models fail systematically on long-horizon manipulation tasks despite strong short-horizon performance. We show that this failure is not resolved by extending context length alone in the current reactive execution setting; instead, it stems from three recurring execution-loop deficiencies: the memory gap, the verification gap, and the recovery gap. We present HELM, a model-agnostic framework that addresses these deficiencies with three components: an Episodic Memory Module (EMM) that retrieves key task history via CLIP-indexed keyframes, a learned State Verifier (SV) that predicts action failure before execution from observation, action, subgoal, and memory-conditioned context, and a Harness Controller (HC) that performs rollback and replanning. The SV is the core learning contribution: it consistently outperforms rule-based feasibility checks and ensemble uncertainty baselines, and its effectiveness depends critically on access to episodic memory. On LIBERO-LONG, HELM improves task success rate by 23.1 percentage points over OpenVLA (58.4% to 81.5%), while extending the context window to H=32 yields only a 5.4-point gain and same-budget LoRA adaptation remains 12.2 points below HELM. HELM also improves long-horizon performance on CALVIN and substantially boosts recovery success under controlled perturbations. Ablations and mechanism analyses isolate the contribution of each component, and we release LIBERO-Recovery as a perturbation-injection protocol for evaluating failure recovery in long-horizon manipulation.

### 1. 核心问题与整体含义（研究动机和背景）
现有的视觉-语言-动作（VLA）模型（如 OpenVLA, Octo）在短程任务中表现优异，但在长程操控任务中成功率大幅下降。作者通过实验发现，这种失效并非单纯因为上下文窗口（Context Window）不够长，而是源于当前“反应式执行”模式下的三个结构性缺陷：
*   **记忆鸿沟 (Memory Gap, $F_M$)**：固定窗口会丢弃已完成子目标的证据，导致模型重复执行已完成的动作。
*   **验证鸿沟 (Verification Gap, $F_V$)**：模型直接执行生成的动作，缺乏对物理可行性或语义正确性的预执行检查。
*   **恢复鸿沟 (Recovery Gap, $F_R$)**：一旦动作失败，模型会在错误状态上继续尝试后续步骤，导致级联失效。

### 2. 方法论：核心思想与关键技术
HELM 提出了一个模型无关的框架，通过三个核心组件增强 VLA 的执行循环：
*   **情节记忆模块 (EMM)**：建立一个基于 CLIP 嵌入索引的关键帧存储库。它在子目标完成、检测到失败或每隔 20 步时写入关键帧。在执行时，通过余弦相似度检索最相关的历史上下文，并将其序列化为文本注入 VLA 的语言输入中。
*   **状态验证器 (SV) - 核心贡献**：这是一个轻量级的 MLP 模块，用于在动作执行前预测失败概率 $p_{fail}$。其输入包括当前观测、检索到的记忆上下文、拟执行动作和当前子目标。它能识别出那些看似合理但在特定任务历史背景下是错误的动作。
*   **Harness 控制器 (HC)**：负责协调整个循环。如果 SV 预测失败概率超过阈值，HC 会触发恢复机制：从记忆中检索最近的成功检查点，引导模型回滚（Rollback）到该状态并重新规划。对于不可逆环境，也提供了前向恢复（Forward Recovery）变体。

### 3. 实验设计
*   **数据集/场景**：
    *   **LIBERO-LONG**：包含 10 个长程任务，平均 5.8 个子目标。

    *   **LIBERO-Recovery (新提出)**：专门用于评估故障恢复能力的协议，通过在随机子目标边界注入物体位移或夹具状态翻转等扰动。
*   **对比方法**：包括 OpenVLA (基准)、延长上下文长度 (H=32/64)、Oracle 记忆、基于规则的验证器、集成不确定性 (Ensemble)、同等数据预算下的 LoRA 微调、以及 Reflexion (自我反思) 等 9 个基线。

*   **推理开销**：每步增加约 **12 毫秒** 的延迟，相比原始 VLA 模型仅增加约 15% 的计算开销。

### 5. 实验数量与充分性
*   **实验规模**：在两个主流基准和一个新协议上进行了测试，每个实验均运行了 3 个随机种子并报告了标准差。
*   **消融实验**：非常充分。作者分别验证了 EMM、SV、回滚机制的独立贡献，并对比了不同的检索策略（随机、最近、CLIP、学习型）和失败预测的时间跨度。
*   **客观性**：通过引入“同等数据预算下的 LoRA 微调”作为对比，有力地证明了 HELM 的提升并非仅仅来自额外的训练数据，而是来自其结构化的验证和恢复机制。

### 6. 主要结论与发现
*   **长上下文并非万能**：将上下文窗口从 8 增加到 32 仅提升了 5.4% 的成功率，而 HELM 提升了 23.1%。
*   **HELM 表现卓越**：在 LIBERO-LONG 上将 OpenVLA 的成功率从 58.4% 提升至 **81.5%**。
*   **SV 的关键性**：学习到的状态验证器优于基于规则的检查和集成方法，且其性能高度依赖于记忆上下文。
*   **模型无关性**：HELM 在 Octo 模型上也取得了类似的显著提升（+21.6%），证明了其通用性。

### 7. 优点
*   **结构化解决失效**：针对长程任务的三个痛点（记忆、验证、恢复）给出了清晰且模块化的解决方案。
*   **预执行验证**：相比于事后反思（Post-hoc reflection），预执行预测能有效防止错误状态的传播。
*   **高效性**：SV 模块非常轻量，在几乎不增加推理负担的情况下显著提升了鲁棒性。
*   **新评估协议**：提出的 LIBERO-Recovery 为机器人领域研究故障恢复提供了标准化的工具。

### 8. 不足与局限
*   **数据依赖性**：SV 的训练需要针对特定环境收集约 5 万步的 rollout 数据，这限制了其在完全零样本（Zero-shot）场景下的部署。
*   **回滚假设**：回滚机制假设环境动作在某种程度上是可逆的或可以重新定位的，虽然提出了前向恢复变体，但在极端不可逆任务中的表现仍有待提升。
*   **Sim-to-Real 挑战**：所有实验均在仿真环境中完成，真实世界中的传感器噪声和视觉偏移可能会影响 CLIP 检索和 SV 的准确性。
*   **子目标分解依赖**：虽然影响较小，但框架仍依赖于 VLA 初始生成的子目标分解质量。
